# 幻灯片模板选择指南

不同信息有不同的最佳呈现方式。选对模板，读者一眼就懂。

---

## Default — 通用要点页

**适合：** 概念解释、问题定义、特性总结、优缺点

**为什么：** 人阅读时视线自然从上到下，图标抓眼球，标题定调，要点逐条消化，底部提示收尾。最通用、最不容易出错。

**示例：**
- "DQN 要解决什么问题"
- "Transformer 的核心组件"
- "这个实验的局限性"

```
🧠 DQN 要解决什么问题                    2 / 5
  ◆ 传统 Q-learning 无法处理高维状态
  ◆ 需要函数近似器来泛化
  ◆ 深度学习提供特征提取能力
  ┌─────────────────────────────┐
  │ 💡 DQN 是第一个将 DL 与 RL  │
  │     结合的工作              │
  └─────────────────────────────┘
```

---

## Compare — 左右对比

**适合：** 算法对比、方案取舍、前后对比、A/B 分析

**为什么：** 人眼天生擅长左右扫视找差异。把两个东西放在同一视线层级，差异一目了然。适合 2~6 条对比点，再多就变成列表了。

**示例：**
- "DQN vs 传统 Q-learning"
- "CNN vs Transformer"
- "训练前 vs 训练后的表现"

```
            DQN vs 传统 Q-learning              3 / 5
  ┌──────────────┐    ┌──────────────┐
  │  Q-learning   │    │     DQN      │
  │  • 查表法     │    │  • 神经网络  │
  │  • 状态有限   │    │  • 泛化能力强 │
  │  • 收敛慢     │    │  • 稳定收敛  │
  └──────────────┘    └──────────────┘
```

## Data — 数据/指标展示

**适合：** 实验结果、性能指标、统计数字、里程碑数据

**为什么：** 人对大数字和对比最敏感。去掉多余文字，把核心数字放大，配合简短标签，信息密度最高。适合展示 2~4 个关键数字。

**示例：**
- "DQN 在 Atari 上的表现"
- "模型参数量对比"
- "训练耗时统计"

```
      DQN 在 Atari 上的表现                 4 / 5
      ┌──────────┐  ┌──────────┐
      │  29/49   │  │  750%    │
      │ 超越人类  │  │ 平均得分 │
      ├──────────┤  ├──────────┤
      │  100%    │  │  1 套    │
      │ 仅像素输入│  │ 统一架构 │
      └──────────┘  └──────────┘
```

## Quote — 论文引用/名言

**适合：** 原始论文定义、经典论断、核心定理

**为什么：** 引文需要与正文区隔。加大字号、加引号装饰、标注出处，让读者知道"这是原话"。适合整页只放一段话的场景。

**示例：**
- 论文中的关键定义
- 大佬的经典观点
- 某篇工作的核心结论

```
            "
  We propose a new family of policy
  gradient methods for reinforcement
  learning, which alternate between
  sampling data through interaction
  with the environment, and optimizing
  a 'surrogate' objective function...
            "
        — John Schulman et al.
          PPO 论文 (2017)
```

## Code — 代码展示

**适合：** 核心算法伪代码、关键实现片段、API 使用示例

**为什么：** 代码需要等宽字体、语法区分、足够宽的背景框。单独抽出来作为一页，比插在正文里更清晰。

**示例：**
- "PPO Clipping 实现"
- "损失函数定义"
- "数据预处理流程"

```
        PPO Clipping 核心实现               3 / 5
  ┌────────────────────────────────┐
  │  python                        │
  │                                │
  │  ratio = torch.exp(...)        │
  │  surr1 = ratio * advantage     │
  │  surr2 = torch.clamp(...)      │
  │  loss = -torch.min(surr1,       │
  │                  surr2).mean()  │
  └────────────────────────────────┘
```

## 选型速查表

| 你要呈现的内容 | 推荐模板 | 理由 |
|---|---|---|
| 概念解释、问题定义 | `default` | 层次清晰，逐条消化 |
| A vs B 对比 | `compare` | 左右扫视，差异直观 |
| 实验数据、统计指标 | `data` | 数字放大，冲击力强 |
| 论文原话、经典定义 | `quote` | 与正文区隔，突出出处 |
| 算法代码、实现片段 | `code` | 等宽排版，语法清晰 |

---

## 怎么用

在数据文件的 `slides` 数组里加 `type` 字段：

```js
slides: [
  { type: "default", icon: "🧠", title: "...", points: [...], tip: "..." },
  { type: "compare", title: "...", leftTitle: "A", leftPoints: [...], rightTitle: "B", rightPoints: [...] },
  { type: "data", title: "...", stats: [{ value: "89%", label: "准确率" }] },
  { type: "quote", quote: "...", author: "...", source: "..." },
  { type: "code", language: "python", code: "..." },
]
```

不知道用哪个 → 用 `default`，不会错。
